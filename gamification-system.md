# Gamification & Reward System

## Video Game Time Rewards 🎮

### Base Reward Structure

#### Per Question Performance
| Performance | Video Game Minutes | Bonus |
|-------------|-------------------|-------|
| First attempt correct | 5 minutes | - |
| Second attempt correct | 3 minutes | - |
| Third attempt correct | 1 minute | - |
| Fourth+ attempts | 0 minutes | Try reviewing the concept! |

#### Streak Bonuses
| Streak Length | Bonus Minutes | Total per Question |
|---------------|---------------|-------------------|
| 3 in a row | +1 minute | 6 minutes |
| 5 in a row | +2 minutes | 7 minutes |
| 10 in a row | +5 minutes | 10 minutes |
| 20 in a row | +10 minutes | 15 minutes |

## Penalties & Safeguards

### Time Deductions
| Action | Penalty | Reason |
|--------|---------|--------|
| Skip without attempt | -2 minutes | Encourage trying |
| Hint used | -1 minute per hint | Balance challenge |
| Multiple wrong answers | Cap at 0 minutes | No negative time |

### Safeguards
- **Minimum daily cap**: Can earn at least 30 minutes/day
- **Maximum daily cap**: Can earn up to 180 minutes/day (3 hours)
- **Rollover**: Unused time expires after 7 days
- **Redemption**: Must redeem in 30-minute blocks
- **Parent override**: Parents can adjust caps/rules

---

## Redemption Options

### How to Use Earned Time

1. **Immediate Play**
   - Redeem in 30-min blocks
   - Use same day only

2. **Bank for Later**
   - Save up to 7 days
   - Combine for longer sessions

3. **Weekend Bonus**
   - Weekend redemptions get +25% time
   - 60 minutes → 75 minutes

4. **Trade-In Options**
   - 120 minutes → 1 new game rental
   - 300 minutes → 1 new game purchase credit
   - 60 minutes → Extra 30 min past bedtime

## Implementation Notes

### Technical Requirements
- Persistent storage of user progress
- Real-time time bank updates
- Streak tracking across sessions
- XP and level calculations
- Leaderboard refresh (daily/weekly)
- Parent portal with analytics

- **Frontend**: React/Next.js for dashboard
- **Backend**: Node.js/Express for API
- **Database**: PostgreSQL for user data
- **Cache**: Redis for leaderboards

## Testing & Balancing

### Phase 1: Beta Testing
- Start with 10 students
- Monitor time earned vs study time
- Adjust caps if too easy/hard to earn time

### Phase 2: Balance Tuning
- Target: 30-45 min study → 30-60 min game time
- Adjust multipliers based on engagement
- Survey students on fairness

### Phase 3: Long-term Sustainability
- Ensure system remains motivating after 3+ months
- Introduce seasonal events
- Add new challenges and badges
- Keep rewards fresh and exciting
